# Отчёт: Паттерн Composite

**Проблема**: как сделать единый интерфейс для управления сложной системой в качестве умного дома, 
где нужен контроль над одним устройством, и группой.

---

## Почему нужен паттерн Composite
Для разработки системы в которой будет возможность:
- Включать/выключать **отдельные устройства** (лампа, кондиционер)
- Управлять **группами** (все устройства в комнате)
- Управлять **всем домом** одной командой
- Организовать древовидную структуру: отдельные устройства → комнаты → дом
- Легко **добавлять новые типы устройств**

### Без паттерна (проблемы):
- Разные методы для разных уровней
- логика повторяется
- расширение неоптимизировано
### Вывод
Паттерн Composite дает плюсы:
 Предметная область иерархична (дом → комнаты → устройства)
 Единый интерфейс упрощает клиентский код
 Рекурсивный обход позволяет управлять всей структурой одной командой
 Легко расширять: новое устройство = новый класс, наследующий Device
 Соблюдаются принципы SOLID
